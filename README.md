# dotfiles

Hyprland desktop config: window manager (Lua-based config), waybar, wofi,
kitty, mako, plus the custom keybind/appearance GUI editor used to manage it.

## Install

```sh
git clone https://github.com/<your-username>/dotfiles.git ~/dotfiles
~/dotfiles/install.sh
```

On openSUSE, this first installs every package these dotfiles expect via
`zypper` (prompts for your sudo password once). On any other distro that
step is skipped with a note - see Prerequisites below and install them by
hand with your own package manager.

On openSUSE it also builds the [Graphite GTK
theme](https://github.com/vinceliuice/Graphite-gtk-theme) (green accent,
black tweak) from source into `~/.cache/dotfiles-vendor/`, since it isn't
packaged, and sets Papirus's folder color to green. See GTK theme below.

It then symlinks everything under `.config/` and `.local/bin/` into the
matching path under `$HOME` (backing up any existing file as `.bak`), then
regenerates `keybinds.lua` / `appearance.lua` / `monitors.lua` from their
`.json` sources. Re-running it is safe.

Monitor layout (`monitors.json`) is the one file that's **not** symlinked -
it's seeded with a generic auto-detect default on first install, but stays
a real per-machine file after that, since output names/positions differ
per box. After Hyprland is running, press `SUPER+SHIFT+E` to open the
config editor, go to the Monitors tab, and click "Detect connected
monitors" to fill in the real layout for that machine.

## GTK theme

`.config/gtk-3.0/settings.ini` and `.config/gtk-4.0/settings.ini` are
tracked and symlinked like everything else, pointing at
`gtk-theme-name=Graphite-green-Dark` / `gtk-icon-theme-name=Papirus-Dark`.
`install.sh` builds the Graphite theme (`-t green -c dark --tweaks black
-l`) and colors Papirus's folders green so this actually resolves to
something on a fresh machine. On non-openSUSE boxes, install
`papirus-icon-theme` + `papirus-folders` by hand, clone
[Graphite-gtk-theme](https://github.com/vinceliuice/Graphite-gtk-theme) and
run its `install.sh` with the same flags, then run `papirus-folders -C
green -t Papirus-Dark`.

## Prerequisites

Core:
- `hyprland`, `kitty`, `waybar`, `wofi`, `mako`, `hypridle`, `hyprlock`,
  `hyprpaper`, `waypaper`, `thunar`
- `wireplumber` / `pipewire` / `pipewire-pulse`, `brightnessctl`,
  `playerctl`
- `python3` with Tk (`python3-tk` / `python313-tk`) - needed for the
  `SUPER+SHIFT+E` config editor GUI
- `papirus-icon-theme`, `papirus-folders`, `sassc`, `git` - GTK theme (see
  below)

Used by specific keybinds (edit `keybinds.json` to drop any you don't
want):
- `grim`, `slurp`, `swappy` - screenshots
- `wl-clipboard`, `cliphist` - clipboard history
- `pavucontrol`, `nm-connection-editor`, `blueman-manager` - float-only
  window rules assume these
- `yazi`, `xarchiver`, `google-chrome` - launched by specific binds
- [`swayosd`](https://github.com/ErikReider/SwayOSD) - on-screen volume/
  brightness indicator; `.local/bin/hypr-osd.sh` falls back to plain
  `wpctl`/`brightnessctl` if it isn't installed, so it's optional
- [`nwg-displays`](https://github.com/nwg-piotr/nwg-displays) - visual
  drag-and-drop monitor arrangement tool; pairs with the config editor's
  "Detect connected monitors" button to persist a layout you arrange with it

## Structure

```
.config/hypr/
  hyprland.lua        hand-written: autostart, window rules, look and feel
  keybinds.json        \_ edited via SUPER+SHIFT+E, or by hand
  appearance.json      /
  monitors.json         not tracked as "yours" - see Install above
  keybinds.lua, appearance.lua, monitors.lua, keybinds.txt
                        generated by install.sh / the editor - gitignored
  hyprlock.conf, hyprpaper.conf, hypridle.conf
.config/waybar/, .config/wofi/, .config/kitty/, .config/mako/
.config/gtk-3.0/settings.ini, .config/gtk-4.0/settings.ini
  theme + icon theme names - see GTK theme above
.local/bin/
  hypr-config-editor.py, hypr_config_lib.py   the SUPER+SHIFT+E GUI editor
  hypr-osd.sh, identify-osd.sh, powermenu.sh, show-keybinds.sh
```
